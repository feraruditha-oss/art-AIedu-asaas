# 仓库文档上传说明

这些文档建议放进 GitHub 仓库，用来沉淀产品定位、开发方向、原型说明和后续迭代依据。

## 建议上传结构

```text
art-AIedu-asaas/
├── README.md
└── docs/
    ├── 00-repository-upload-guide.md
    ├── prd.md
    ├── mvp.md
    ├── user-flow.md
    ├── prototype-pages.md
    ├── development-issues.md
    ├── acquisition-entry-system.md
    ├── assessment-model.md
    ├── child-assessment-report-template.md
    ├── content-engine.md
    ├── content-growth-engine.md
    ├── content-library-seed.md
    ├── platform-content-benchmark.md
    ├── data-collection-template.md
    └── interview-script.md
```

## 每份文档的作用

- `README.md`：项目总说明，别人打开仓库第一眼看到的内容。
- `prd.md`：产品需求文档，说明系统要解决什么问题、给谁用、核心功能是什么。
- `mvp.md`：第一阶段最小可行产品范围，防止一开始做得太散。
- `user-flow.md`：家长、老师、机构老板从获客到预约的完整路径。
- `prototype-pages.md`：原型页面说明，方便后续开发照着页面做。
- `development-issues.md`：开发任务清单，适合后面拆成 GitHub Issues。
- `acquisition-entry-system.md`：多渠道获客入口设计。
- `assessment-model.md`：少儿艺术测评模型。
- `child-assessment-report-template.md`：少儿测评报告内容模板。
- `content-engine.md`：内容生成模块的基础设计。
- `content-growth-engine.md`：内容增长引擎设计，重点是平台内容、转化和本地招生。
- `content-library-seed.md`：动态内容库种子，包含热点、节日、自我迭代和原创去重机制。
- `platform-content-benchmark.md`：小红书、抖音、视频号内容规律参考。
- `data-collection-template.md`：真实学生作品和机构数据采集模板。
- `interview-script.md`：访谈机构校长和老师时使用的问题清单。

## 上传建议

先上传 `README.md` 和整个 `docs/` 目录即可。  
`prototype/` 原型页面可以之后单独上传，或放在仓库根目录下的 `prototype/` 目录。
